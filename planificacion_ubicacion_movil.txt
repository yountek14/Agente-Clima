Proyecto: Agente-Clima
Fecha: 16/06/2026
================================================================================

OBJETIVO
--------
Cuando un usuario entre al sitio desde un celular, detectar su ubicacion
mediante el GPS del dispositivo y auto-seleccionar la comuna mas cercana de
la Region de Los Lagos. Sin depender de APIs externas de pago (Google Maps).


ENFOQUE
-------
- API nativa del navegador: navigator.geolocation.getCurrentPosition()
- Calculo de distancia haversine en el backend (matematica pura)
- Comparacion contra las 15 comunas existentes en COMUNAS_DE_LOS_LAGOS
- Sin Google Maps API key, sin nuevas dependencias en requirements.txt


ARQUITECTURA
------------

[Celular] -- GPS --> [navigator.geolocation] -- lat/lon --> [POST /api/geolocalizar]
                                                                      |
                                                          Haversine contra 15 comunas
                                                                      |
                                                          Comuna mas cercana (< 100km)
                                                                      |
                                                   { comuna_id, nombre, distancia_km }
                                                                      |
                                          [app-main.js] --> seleccionarComuna(id, nombre)
                                                                      |
                                          Ya existente: actualiza UI, pronostico, chat


CAMBIOS POR ARCHIVO
-------------------

1. app.py - NUEVO ENDPOINT: POST /api/geolocalizar
   - Recibe JSON: { "lat": float, "lon": float }
   - Calcula distancia haversine a cada una de las 15 comunas
   - Distancia <= 100km: devuelve { "comuna_id", "nombre", "distancia_km" }
   - Distancia > 100km: devuelve { "comuna_id": null, "mensaje": "fuera de la region" }
   - Formula haversine:
     a = sin²(Δlat/2) + cos(lat1)*cos(lat2)*sin²(Δlon/2)
     c = 2*atan2(√a, √(1-a))
     d = 6371 * c

2. app-main.js - NUEVA FUNCION: detectarUbicacion()
   - Verifica si navigator.geolocation esta disponible
   - Llama a getCurrentPosition() con timeout de 10s
   - Envia coordenadas a POST /api/geolocalizar
   - Si recibe comuna_id valido, llama a seleccionarComuna(id, nombre)
   - Si falla o es denegado, no hace nada (silent fail)
   - Se llama automaticamente al final de inicializarAplicacion()
   - Se expone como window.detectarUbicacion para el boton manual

3. index.html - BOTON DE UBICACION
   - Dentro de .ubicacion-selector-footer, anadir un span con el icono 📍
   - onclick="detectarUbicacion()"
   - Visible tanto en escritorio como movil
   - Sirve como gatillo manual si el usuario denego el permiso inicial


FLUJO DE USUARIO EN MOVIL
-------------------------

1. Usuario abre agente-clima.xyz desde su celular
2. Navegador muestra: "Este sitio quiere conocer tu ubicacion" [Permitir|Bloquear]
3. PERMITE:
   a. Se obtienen coordenadas GPS
   b. Backend calcula comuna mas cercana
   c. Se auto-selecciona la ciudad en el selector
   d. El pronostico semanal se actualiza
   e. El chat ya referencia la ciudad correcta
4. BLOQUEA:
   a. Se queda en Puerto Montt (default)
   b. Puede usar el selector manual de ciudades
   c. Puede tocar el boton 📍 para reintentar

CASOS EDGE
----------
| Escenario                                  | Comportamiento                        |
|--------------------------------------------|---------------------------------------|
| Usuario en Santiago (>900km)               | API devuelve null, no se cambia ciudad|
| Usuario en Castro con GPS                  | Auto-detecta Castro                   |
| Usuario niega permiso de ubicacion         | Default Puerto Montt, boton 📍 activo |
| Navegador sin geolocation API (HTTP, etc.) | No se ejecuta, sin errores            |
| Usuario sin GPS (desktop sin chip GPS)     | Geolocation falla, no hace nada       |
| GPS devuelve coordenadas inexactas         | Se elige la mas cercana igual         |
