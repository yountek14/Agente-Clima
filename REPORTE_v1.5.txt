  1. CAMBIOS ESTRUCTURALES Y DE ARQUITECTURA
================================================================================

  ANTES (v1.0):
  - app.py servía index.html como archivo estático (send_from_directory)
  - Sin templates/ (solo static/index.html)
  - Sin caché de clima ni historial de chat en servidor

  DESPUÉS (v1.5):

  - Endpoints REST: /api/comunas, /api/pronostico, /api/chat, /generar-reporte

  - Memoria efímera HISTORIAL_CHAT por session_id (se pierde al recargar)
  - Inyección de contexto climático en vivo al prompt del chatbot
  - Flask-CORS configurado para peticiones frontend

================================================================================
  3. BACKEND — SERVIDOR FLASK (app.py)
================================================================================

  3.1 ENDPOINTS
  ─────────────
  GET  /                 → index.html (Jinja2 renderizado)
  GET  /api/comunas      → Lista de comunas {id, nombre}
  GET  /api/pronostico   → Pronóstico Open-Meteo con caché de 10 min
  POST /api/chat         → Chatbot con historial por sesión
  POST /generar-reporte  → Pipeline de 6 pasos + envío por correo

  3.2 CACHÉ INTELIGENTE
  ─────────────────────
  CACHE_CLIMA = {}  # comuna_id → {timestamp, data}
  - Expira a los 10 minutos
  - Evita saturar la API de Open-Meteo
  - Se usa también como contexto para el chatbot

  3.3 CHATBOT
  ───────────
  HISTORIAL_CHAT = {}  # session_id → [{role, content}]
  - Memoria volátil (se pierde al reiniciar Flask)
  - Session ID generado aleatoriamente por el frontend
  - System prompt inyectado con:
      * Nombre de comuna seleccionada
      * Datos climáticos en vivo desde caché
  - Usa agente.llm.invoke() con LangChain

  3.4 PIPELINE DE REPORTE
  ───────────────────────
  Endpoint /generar-reporte:
  1. Recibe correo + comuna_id
  2. Llama a agente.generar_reporte() con parámetros completos
  3. Espera retorno con formato {exito, html_reporte, pasos_ejecutados}
  4. Devuelve vista previa HTML al frontend


================================================================================
  4. CORRECCIONES DE BUGS (vs v1.0)
================================================================================

  BUG #1: CSS no cargaba con Go Live
  ──────────────────────────────────
  Síntoma: HTML mostraba solo estructura básica
  Causa:  Jinja2 {{ url_for() }} no se procesa con Live Server
  Fix:    Ejecutar con .\.venv\Scripts\python.exe app.py
          y abrir http://localhost:5000

  BUG #2: Nombre incorrecto de archivo CSS
  ────────────────────────────────────────
  Síntoma: CSS no cargaba
  Causa:  HTML referenciaba "styles.css" pero el archivo es "style.css"
  Fix:    Cambiado a "style.css"

  BUG #3: Argumentos incorrectos en generar_reporte()
  ──────────────────────────────────────────────────
  Síntoma: Endpoint /generar-reporte devolvía 500
  Causa:  app.py pasaba (comuna_id, correo_destinatario)
          pero agente.py esperaba (email, comuna_id, nombre_comuna, latitud, longitud)
  Fix:    Corregida la llamada con todos los parámetros desde COMUNAS_DE_LOS_LAGOS

  BUG #4: agente.generar_reporte() retornaba formato incorrecto
  ──────────────────────────────────────────────────────────────
  Síntoma: Frontend recibía error aunque el pipeline funcionaba
  Causa:  Retornaba {comuna, clima, conclusion, ...}
          app.py esperaba {exito, html_reporte, pasos_ejecutados}
  Fix:    Retorno cambiado a {exito: True, html_reporte, pasos_ejecutados}

  BUG #5: Overflow en flexbox del chat
  ────────────────────────────────────
  Síntoma: Input y botones desaparecían al enviar mensajes
  Causa:  Falta min-height: 0 en flex items con overflow
  Fix:    Agregado min-height: 0 + flex-shrink: 0 donde corresponde

  7. CÓMO EJECUTAR EL PROGRAMA Y RUTA LOCAL
================================================================================

  OPCIÓN 1 — Script de inicio automático (recomendado):
  ─────────────────────────────────────────────────────
  .\iniciar.ps1
  (Ejecuta el servidor Flask y muestra las URLs activas)

  OPCIÓN 2 — Comando manual directo:
  ──────────────────────────────────
  .\.venv\Scripts\python.exe app.py

  RUTA LOCAL EN EL NAVEGADOR:
  ───────────────────────────
  http://localhost:5000

  NOTAS:
  - El puerto por defecto es 5000.
  - Para cambiar el puerto: $env:PORT=8080; .\.venv\Scripts\python.exe app.py
  - Flask sirve tanto el frontend (templates/index.html) como la API REST
    desde la misma URL. NO se debe usar Live Server de VS Code.
  - Asegurarse de tener el entorno virtual (.venv) con las dependencias
    instaladas: .\.venv\Scripts\pip.exe install -r requirements.txt
